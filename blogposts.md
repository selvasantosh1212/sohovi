## 3. How to Write a Sohovi Blog Post

Every post published on the Sohovi blog must follow this guide. This is the single source of truth for writers, contractors, and contributors.

---

### 3.1 The Conversion Anatomy (7 Steps — Always in This Order)

A post converts when it does these 7 things in this order. Skip any one and the post underperforms.

**Step 1 — Answer in the first 80 words**

Give the searcher the direct, useful answer immediately. Don't save it for the middle or end. This wins featured snippets, AI Overviews, and earns the trust that makes the reader continue.

*Example — "How to Remove Duplicate Rows in Excel":*
> "The fastest fix: Select your data → Data tab → Remove Duplicates → pick which columns to check → click OK. This takes under 30 seconds and works for clean, consistent data. For fuzzy matches, case variants, or trailing-space duplicates, read on — Excel's built-in tool misses all of those."

**Step 2 — Show the manual/free method honestly**

Teach the Excel / Sheets / free-tool method completely. Don't cripple it to push Sohovi — visitors detect that, and Google penalizes thin content. A reader should be able to complete the task after reading this section, even if they never sign up.

**Step 3 — Surface the manual method's breaking point**

Every manual method has one genuine failure mode. Find it, state it factually, show a concrete example. This is the conversion pivot — it must be TRUE, not manufactured.

Examples:
- Excel Remove Duplicates: "0 duplicate values found and removed" on data with trailing spaces
- Power Query date import: silent date swap when regional settings differ
- Manual phone standardization: ambiguous country codes for numbers without prefixes

**Step 4 — Demo at the breaking point**

At the moment the manual method fails, show Sohovi solving it. Use `[IMAGE: description]` placeholders — the diagram system maps these to SVGs automatically. One concrete visual demo outconverts five paragraphs of product copy.

**Step 5 — Free tool link**

Every post related to one of the 9 free tools must embed or link to it:

| Tool | URL |
|------|-----|
| Duplicate Row Remover | `/tools/remove-duplicates` |
| CSV Merger | `/tools/csv-merger` |
| CSV Column Picker | `/tools/csv-columns` |
| CSV → JSON | `/tools/csv-to-json` |
| JSON → CSV | `/tools/json-to-csv` |
| CSV → SQL | `/tools/csv-to-sql` |
| CSV → Markdown | `/tools/csv-to-markdown` |
| Test Data Generator | `/tools/test-data-generator` |
| Excel Formula Explainer | `/tools/formula-explainer` |

**Step 6 — Topic-specific CTA at end**

Formula: *"[Do the exact task this post is about] in Sohovi — [specific limitation removed]. Try it free on your own file."*

❌ **Never use generic boilerplate:** "See what's wrong with your data — try Sohovi free"

✅ **Always be specific:**
- "Catch the duplicates Excel can't see — paste your data into our free duplicate remover (runs in your browser, nothing uploaded)."
- "Standardize a thousand phone numbers in one click — and get a report of the ones that need a human look. Upload your export free."
- "Not sure if your dates are silently corrupted? Profile the file in Sohovi — mixed formats light up instantly."

**Step 7 — FAQ section (3–5 questions)**

Pull from Google's "People Also Ask" for the target keyword. Format:

```markdown
## Frequently Asked Questions

**Q: [question]**
[1–3 sentence answer]
```

### 3.4 Internal Linking Rules

Every post links in three directions:

**UP to the hub** — the cluster's pillar or hub page:
- Data cleaning posts → `/blog/data-cleaning-complete-process`
- DQ dimensions posts → `/blog/data-quality-six-dimensions-hub`
- Migration posts → `/blog/data-quality-checklist-data-migration`
- Tool posts → `/tools` (tools hub)
- Business case posts → `/blog/bad-data-cost-small-business`

**ACROSS to 2–3 siblings** — closely related posts at the same level.

**DOWN to one free tool** — the relevant tool from the table in Step 5.

**No orphans.** When you add a new post, also update at least one existing post to link to it (add a `[INTERNAL LINK: ...]` line in that post and re-seed).

The automatic link injection system (`lib/blog-internal-links.ts`) handles first-occurrence linking for ~90 known terms at render time. You don't need to manually hyperlink these terms — but do add new terms to that file if your post introduces a concept that should always link somewhere.

**2. Inline Sohovi mentions** (`injectInlineSohoviMentions`): Problem-signal keywords trigger a short branded sentence after a paragraph. Fires up to 3× per post, starting from the 3rd paragraph, min 3 paragraphs between fires.

### 3.7 Seeding Workflow: MD File → Live Post

```bash
# 1. Create the .md file
touch app/blog/posts/your-slug.md
# (write the content)

# 2. Seed to Supabase (upserts on slug — safe to re-run)
npm run seed:md

# 3. Verify locally
npm run dev
# visit http://localhost:3000/blog/your-slug

# 4. Commit and push
git add app/blog/posts/your-slug.md
git commit -m "feat: add [topic] blog post"
git push origin main
```

**What the seed script does:**
- Reads frontmatter: title, slug, category, supportingKeywords, seo_title, seo_description
- Auto-computes: excerpt (first paragraph), read_time_min (÷200 WPM), published_at (NOW)
- Sets `published: true` — all seeded posts go live immediately
- Upserts on slug — re-seeding updates existing content

**Stagger publish dates:** After a batch seed, all posts get the same timestamp. To spread dates (avoid same-day bulk drops), manually update `published_at` in Supabase, or use `scripts/stagger-dates.ts`.

- [ ] No broken image paths — use `[IMAGE: description]` format only, never `![](broken.png)`

### 3.9 Publishing Strategy

1. **Priority order:** 1–2 first (BOFU converts immediately at low volume) → 3–4 (volume) → 5–11.
2. **Cadence:** 10–15 posts per week max, staggered publish dates. Never a same-day bulk drop of 20+ posts.
3. **Every post must:** link UP to hub, link to one free tool, end with topic-specific CTA.
4. **Hub pages to maintain:** `data-cleaning-complete-process` and `data-quality-six-dimensions-hub` are the two active hubs. Every new post in those clusters must link back to them.
5. **Measure:** track signups by landing post in analytics. Double down on the cluster that converts, not just the one that ranks.
